# ========================================
#  收集
#  >收集任何需要关注的事件
#  整理
#  > 既不需要处理，又无保留价值-->丢弃
#  > 不需要处理，但有保留价值-->归档
#  > 需要处理的信息-->创建任务
#  评估可执行度
#  >执行度从0分~5分，0分基本不可执行，5分可立即执行。完全不可执行为“激励”。
#  >可在2分钟内完成，基本上都为5分。
#  组织
#  >创建任务时，要有一个快速的判断，包括：
#  >what（做什么）
#  >who（谁来做）
#  >when（什么时候做）
#  >where（什么地点处理）
#  >why（为什么要做，可以用原始消息作为原因）
#  >how（如何做）
#  >how much（需要多长时间）
#  >这个判断要非常迅速，并根据判断的结果对任务进行组织，以便后来的处理和查询。
#  >通常按照可以这样的步骤分析：
#   1. 该任务是否属于一个项目？如果是，首先打上项目标记
#   2. 该任务能否在2分钟完成？如果是，马上处理，然后将任务标记为“已完成”
#   3. 根据重要性和紧迫性，为该任务划分优先级
#   4. 该任务是否需要别人处理？如果是，通知对方，然后将任务标记为“等待中”
#   5. 需要自己处理的任务，马上判断处理的地点/场合，并标记
#   6. 对于能够确定时间的任务（日程安排），标记计划处理时间，如果有截止日期，也标记上
#  回顾
#  执行
#  >遵循四个原则：情境是否合适？时间是否够用？精力是否充沛？任务是否重要？
#  >判断任务重要度紧急度：重要/不重要，紧急/不紧急
#    1. 重要且紧急：尽早动手，缓解压力
#    2. 重要不紧急：尽早计划，逐步完成
#    3. 不重要紧急：寻求帮助，设定截止时间
#    4. 不重要不紧急：冷处理
# 每完成一项任务，将该任务标记为“已完成”并归档
